## Remote egress: deferred to 1.3 — orchestrator decision
Rather than spend a second worker spawn building/deploying/exec'ing a throwaway `spike/discordPing` tool to prove remote egress, **1.3's `sync.execute()` IS the proof**. If `worker.sync("projectsFromDiscord")` calls `fetch("discord.com/api/v10/...")` and successfully writes ≥2 rows to the managed Notion DB on first scheduled run, remote egress is empirically proven. If it 4xx/5xx's on the network call, we'll see the failure mode (sandboxed runtime, DNS, TLS, whatever) in the worker run logs.

Cost trade: one spike deploy/exec round ≈ one real sync deploy/exec round. Cheaper to skip the spike.
