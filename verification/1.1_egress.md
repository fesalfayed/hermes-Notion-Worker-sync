# 1.1 Egress Verification

## Test: Outbound HTTPS from Workers runtime to Discord API v10

### Local Exec (standalone tsx — same Node.js runtime as `ntn workers exec --local`)

**Date:** 2026-05-15  
**Command:** `DISCORD_BOT_TOKEN=<redacted> npx tsx src/spike/discordPing.ts`  
**Node.js:** v22.22.2

```
GET https://discord.com/api/v10/channels/000000000000000014
Authorization: Bot <redacted>

Status: 403 Forbidden
Body (first 200 chars): {"message": "Missing Access", "code": 50001}

Egress: YES
```

**Interpretation:** HTTP 403 = the request successfully reached Discord's servers
and was authenticated (the bot token was valid), but channel 000000000000000014
is not visible to the bot (doesn't exist in guild or bot lacks perms).

**Follow-up test with known-good channel (#lobby):**

```
GET https://discord.com/api/v10/channels/000000000000000008
Status: 200 OK
Body: {"id":"000000000000000008","type":0,"name":"lobby","guild_id":"000000000000000007",...}
```

This confirms:
- DNS resolution to discord.com: ✓
- TLS handshake: ✓
- Outbound HTTPS on port 443: ✓
- Discord API v10 reachable: ✓
- Bot token authenticated: ✓ (200 response with channel data)
- Egress from Node.js `fetch()`: **YES**

Note: Original channel ID 000000000000000014 returns 403 because it doesn't
exist in the bot's guild (AGENTIC-AMBLER). Tested with #lobby instead → 200 OK.
